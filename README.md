# MLOps Stock Movement Prediction

This project is an end-to-end MLOps workflow for Indian stock data. It pulls historical OHLCV market data, cleans and stores it in Supabase, builds time-series ML features, performs EDA, trains multiple classifiers with MLflow tracking, saves the best model, and uploads model artifacts back to Supabase Storage.

## What Was Built

The repository has two main workflows:

1. ETL pipeline
   - Pull stock data with `yfinance`.
   - Transform and validate OHLCV records.
   - Add derived market features.
   - Upsert clean rows into a Supabase table.
   - Optionally run the same ETL on a weekday Airflow schedule.

2. ML pipeline
   - Fetch cleaned stock history from Supabase.
   - Build lag, rolling, volatility, and volume features.
   - Create a next-day movement target.
   - Generate EDA plots.
   - Train Logistic Regression, Random Forest, and XGBoost models.
   - Track runs in MLflow.
   - Save the best model locally and upload it to Supabase Storage.

## Supabase Setup

The ETL code expects a `stock_prices` table. The loader checks `symbol` plus `date`, and then upserts on the same pair.

```sql
create table if not exists stock_prices (
    id bigint generated by default as identity primary key,
    symbol text not null,
    date date not null,
    open double precision,
    high double precision,
    low double precision,
    close double precision,
    volume bigint,
    daily_return double precision,
    price_range double precision,
    vwap_approx double precision,
    unique (symbol, date)
);
```

Model training uploads artifacts to a Supabase Storage bucket named `models` and records metadata in a `model_versions` table:

```sql
create table if not exists model_versions (
    id bigint generated by default as identity primary key,
    model_name text not null,
    filename text not null,
    accuracy double precision,
    precision double precision,
    recall double precision,
    f1 double precision,
    roc_auc double precision,
    trained_on timestamp with time zone
);
```

## Step 1: Run The ETL Pipeline

Use the root ETL runner:

```powershell
python main.py
```

`main.py` loads environment variables, creates an `ETLPipeline`, and runs the pipeline for:

```python
RELIANCE.BO
TCS.BO
INFY.BO
```

The ETL flow is:

1. `ETL/pulldata.py`
   - Calls `yf.Ticker(symbol).history(period="5y")`.
   - Returns the downloaded DataFrame in the format expected by the transformer.

2. `ETL/transform.py`
   - Renames yfinance columns to `date`, `open`, `high`, `low`, `close`, and `volume`.
   - Removes duplicate `symbol/date` rows.
   - Drops null OHLCV rows.
   - Drops zero or negative prices.
   - Drops zero-volume rows.
   - Validates OHLC candle consistency.
   - Logs suspicious daily moves greater than 20 percent.
   - Adds `daily_return`, `price_range`, and `vwap_approx`.

3. `ETL/pipeline.py`
   - Checks whether the target symbol/date is already loaded.
   - Fetches existing dates for the symbol from Supabase.
   - Keeps only new rows.
   - Converts pandas timestamps to ISO date strings.
   - Sends records to the loader.

4. `ETL/load.py`
   - Creates a Supabase client.
   - Checks existing rows with `already_loaded`.
   - Upserts records into `stock_prices` with `on_conflict="symbol,date"`.

The returned ETL status is either `inserted` with a row count or `skipped` with a reason such as `already_loaded` or `no_new_data`.

## Step 2: Schedule ETL With Airflow

The Airflow DAG lives in `dags/StockEtlDag.py`.

Important DAG settings:

```text
dag_id: stock_etl_nightly
schedule: 0 3 * * 1-5
meaning: 3:00 AM, Monday through Friday
retries: 2
retry_delay: 5 minutes
```

The DAG dynamically creates one Python task per symbol. Each task calls the async ETL pipeline through `asyncio.run(...)`, logs the result, and pushes the result to Airflow XCom.

To use it, make sure Airflow can see the `dags/` folder and the same `.env` values are available in the Airflow runtime. On Windows, Airflow is usually run through WSL, Docker, or a Linux environment.

Note: `ETL/pulldata.py` documents yfinance BSE symbols as `.BO`, for example `RELIANCE.BO`. If the DAG symbols with `.BSE` return no data, change them to `.BO`.

## Step 3: Run The ML Pipeline

Use the root ML runner:

```powershell
python runml.py
```

`runml.py` currently trains on:

```python
SYMBOL = "RELIANCE.BO"
TABLE_NAME = "stock_prices"
```

The ML flow is:

1. `ML/dataFetch.py`
   - Connects to Supabase.
   - Selects rows from `stock_prices`.
   - Filters by symbol.
   - Sorts by date.

2. `ML/features.py`
   - Adds lag features:
     - `prev_close`
     - `prev_return`
     - `prev_volume`
     - `prev_range`
   - Adds moving averages:
     - `ma_5`
     - `ma_10`
     - `ma_20`
   - Adds price-vs-moving-average signals:
     - `close_vs_ma5`
     - `close_vs_ma10`
     - `close_vs_ma20`
   - Adds volatility features:
     - `volatility_5`
     - `volatility_10`
   - Adds volume features:
     - `volume_ma_5`
     - `volume_ratio`
   - Creates the binary target:
     - `1` when the next close is higher than today's close.
     - `0` otherwise.

3. `ML/eda.py`
   - Logs class balance.
   - Warns when class imbalance is greater than 10 percent.
   - Counts IQR-based outliers.
   - Saves an outlier plot.
   - Saves a feature correlation matrix.
   - Logs the top target-correlated features.

4. `ML/train.py`
   - Splits data chronologically into 80 percent train and 20 percent test.
   - Uses `TimeSeriesSplit` for cross-validation.
   - Uses a scikit-learn `Pipeline` with `StandardScaler` and a classifier.
   - Runs `RandomizedSearchCV` with ROC-AUC scoring.
   - Trains:
     - Logistic Regression
     - Random Forest
     - XGBoost
   - Logs parameters, metrics, model artifacts, and feature importance to MLflow.
   - Selects the best model by test ROC-AUC.
   - Saves `models/best_model.pkl`.
   - Uploads model artifacts and metadata to Supabase.

## Step 5: Model Saving And Upload

After training, the best model is saved as:

```text
models/best_model.pkl
```

The saved pickle bundle contains:

```text
model
feature_cols
model_name
metrics
run_id
```

`ML/train.py` uploads two objects into the Supabase Storage bucket named `models`:

```text
versions/{model_name}_{timestamp}.pkl
best_model.pkl
```

It also inserts the model metrics into the `model_versions` table.

If you only want to upload the current local `models/best_model.pkl`, run:

```powershell
python uploadModel.py
```

## Step 6: Use The Saved Model For Prediction

`ML/predict.py` contains a `Predictor` class. It loads `models/best_model.pkl`, expects a DataFrame containing the same engineered feature columns, and returns:

```text
date
close
signal
probability
```

`signal` is mapped from the model prediction:

```text
1 -> UP
0 -> DOWN
```

## Notes From The Current Codebase

- `README.md` was empty before this rewrite.
- `.env`, `.venv/`, `mlflow.db`, `mlruns/`, `models/`, and EDA plots are ignored by git.
- `runml.py` is the preferred root training entry point because it imports from the uppercase `ML` package.
- `ML/run_training.py` imports from lowercase `ml`, which may be fragile on case-sensitive systems.
- The Airflow DAG text mentions Alpha Vantage, but the active ingestion implementation uses `yfinance`.
- The ETL script uses `.BO` symbols, while the Airflow DAG currently uses `.BSE`; use the suffix that works with yfinance for your target exchange.
